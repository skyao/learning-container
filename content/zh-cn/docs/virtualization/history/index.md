## 1960 年代

### 虚拟化 IBM 大型机

当时 IBM 的程序员 Jim Rymarczyk 开始虚拟化 IBM 大型机。IBM 设计了 CP-40 大型机供内部使用。该系统演变为 CP-67,它使用分区技术同时运行多个应用程序。

## 1990 年代

### java - 进程级虚拟化

1995 年，Sun 发布了 Java 虚拟机（JVM），它允许在单个物理机器上运行多个独立的 Java 应用程序。JVM 是第一个进程级虚拟化技术，它允许在单个物理机器上运行多个独立的应用程序。

在 90 年代初期,Sun Microsystems 提出了 Java,它允许“一次编写,随处运行”范式展开。用户现在可以用 Java 编写一个可以跨各种硬件架构运行的程序。Java 通过引入中间代码(称为 bytecode)来实现这一点,然后可以在跨不同硬件架构的 Java 运行时上执行。这就是 process-level virtualization,Java 运行时环境虚拟化了 POSIX 层。

### vmware - hypervisor

1998 年，VMware 发布了第一个商用虚拟机管理程序，它允许在单个物理机器上运行多个独立的操作系统。VMware 的虚拟机管理程序允许在单个物理机器上运行多个独立的操作系统。

在 1990 年代后期,VMware 介入并推出了自己的虚拟化模型。这与虚拟化实际硬件(如 CPU、内存、磁盘等)有关。这意味着在 VMware 软件(也称为 hypervisor)之上,我们可以运行作系统本身(称为 guests)。这意味着开发人员不仅限于运行 Java 程序,而且可以运行任何要在来宾作系统上运行的程序。

## 2000 年代

### vmware ESX/GSX

2001 年左右,VMware 推出了 ESX 和 GSX 服务器。

GSX 是 Type 2 虚拟机管理程序,因此它需要像 Windows 这样的作系统来运行 guest。

ESX 是 Type 1 虚拟机管理程序,它允许 guest 操作系统直接在虚拟机管理程序上运行。

### vmware vSphere

2003 年,VMware 推出了 vSphere,这是第一个真正的企业级虚拟化平台。它允许用户在单个物理机器上运行多个独立的操作系统。

### 微软 Hyper-V

2008 年,微软推出了 Hyper-V,这是第一个真正的企业级虚拟化平台。它允许用户在单个物理机器上运行多个独立的操作系统。

### 开源虚拟化

2008 年,Xen 项目发布了第一个开源虚拟机管理程序。Xen 是一个 Type 1 虚拟机管理程序,它允许 guest 操作系统直接在虚拟机管理程序上运行。

## 2010 年代

### 微软 Hyper-V 和 Azure

2010 年,微软推出了 Hyper-V,这是第一个真正的企业级虚拟化平台。它允许用户在单个物理机器上运行多个独立的操作系统。

### 微软 Azure

2010 年,微软推出了 Azure,这是第一个真正的企业级虚拟化平台。它允许用户在单个物理机器上运行多个独立的操作系统。
